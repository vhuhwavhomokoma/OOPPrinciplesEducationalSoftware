Static Polymorphism (Compile-Time Polymorphism):
  • This type is resolved during compile time, meaning the decision of which function to call happens before the program runs.
  • Examples include:
    • Function Overloading: Occurs when two or more functions in the same scope have the same name but different parameters (either in number or type).
    • Operator Overloading: Allows existing operators to have new meanings based on the types of operands involved.
  • Static polymorphism is also known as early binding because the method to be executed is determined at compile time.
Dynamic Polymorphism (Run-Time Polymorphism):
  • Dynamic polymorphism is referred to as late binding since the method call is determined during execution, and memory allocation occurs at runtime.
  • This type is resolved during runtime, allowing for more flexible behavior in programs.
  • The primary mechanism here is method overriding, where a subclass provides a specific implementation of a method that is already defined in its superclass.
