What is Encapsulation? Understanding the Basics
Encapsulation is when we:

  • Bundle data (attributes/fields) and actions (methods) together inside a class.

  • Hide the details of how things work inside the class by making some data private. This protects the data from being changed directly from outside the class.

Example:
Imagine a smartphone.

  • You can only interact with it using the screen and buttons (methods).

  • The internal hardware (data) is hidden inside and protected from being damaged.

Key Concepts:

  • Attributes (Data): Information that belongs to an object. For example, a Person object has a name and age.

  • Methods (Actions): Actions that an object can perform. For example, a Person can introduce() themselves.

  • Access Modifiers:
    • Private: Only the class can use this data.
    • Public: Everyone can use this method to interact with the class.
