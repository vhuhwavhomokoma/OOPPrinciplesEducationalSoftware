4. Inheritance

Inheritance is a relationship where one class (subclass) derives properties and behavior (methods) from another class (superclass).

Key Point:

This allows for code reuse and the creation of hierarchical class structures. Itâ€™s often described as an "is-a" relationship.

5. Dependency

A dependency exists when one class uses or depends on another class for its operation. This is typically a temporary relationship where one class relies on another.

Key Point:

Dependency usually involves one class needing another class to perform its function temporarily.

Examples for Dependency and Inheritance

Example of Dependency: A printer class depends on a paper class for its function. Without paper, the printer cannot print.

Example of Inheritance: A dog class can inherit from an animal class. The dog class will inherit properties like "legs" and methods like "eat" from the animal class.
