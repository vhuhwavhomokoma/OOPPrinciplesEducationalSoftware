Diagrams

Diagrams provide a visual way to represent objects and classes. In practice, it’s not necessary to create a highly detailed or artistic drawing every time you want to illustrate a Class or Object, its attributes, and methods—simple, clear representations are ideal. We use diagrams, specifically UML Class diagram and UML Object diagram to represent our ideas of what the Class or Object should look like.

UML Class and Object diagrams are types of structural diagrams. Object diagrams capture a snapshot of an object’s state at a specific moment, while Class diagrams provide a template for how all objects should be structured. Both diagrams are valuable for illustrating how a system interacts with other systems or subsystems, and how data is managed throughout the system.

What is UML?

UML (Unified Modelling Language) modelling is a standardized way to visualize, specify, and document the structure and behaviour of a system. It provides a suite of diagram types, each designed to represent different aspects of the system, such as its structure, interactions, processes, and data flow. Common UML diagrams include Class, Object, Use Case, Sequence, and Activity diagrams, each serving a specific purpose in modelling both the static and dynamic aspects of a system.

Visibility Operators

In UML Class Diagrams, the symbols +, and - represent visibility modifiers or access modifiers for attributes and methods. These symbols indicate the level of access control for each class member:

  • + (Public): The member (attribute or method) can be accessed by anyone.
  • - (Private): The member (attribute or method) can only be accessed within the Class itself.

The reason we have visibility modifiers in the first place is security. If a Person object has an ID attribute, we do not want everyone having the ability to access this attribute. We only want the Person to be able to access the ID. Hence why we make the ID a private member.

Attributes and Methods

As previously explained the attributes are the characteristics of the Class. The main attributes of the Person Class are: name, surname and age.

An attribute in a Class diagram consists of three components:

  • The attribute name.
  • The attribute data type: Determines whether it’s a string, number, or character.
  • Visibility modifier: Determines who can access the attribute.

Methods represent the behaviour or actions of the Class, e.g., the walk method is an action performed by the Person class.

A method is primarily comprised of:

  • Method name
  • Visibility modifier: Determines who can access the method.
  • Visibility modifier: Determines who can access the attribute.
  • Parameters: The values the Method can accept and work with.
  • Return type: What data type the method returns. Please note that void means the method returns nothing.

Getters and Setters

Since attributes are private, it’s important to provide controlled ways to access and modify them. Getters and Setters are methods used for this purpose: getters retrieve (or "get") the values of attributes, while setters allow them to be modified (or "set"). These methods are typically public, defining what the outside world can and cannot do with the class’s attributes, thus enforcing encapsulation and control over the class's data.

In our example, we have getName, getAge and getSurname as getters, and setName, setAge, and setSurname as our getters and setters. These methods operate on the name, age, and surname attributes.
